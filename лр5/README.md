1.Вариант 2. Реализуйте рекурсивную функцию для вычисления степени числа (power(x, n)).
2.Суть алгоритма:
Рекурсивный алгоритм вычисления xn основан на принципе разделяй и властвуй.
Рекурсивное вычисление xn через разбиение задачи на более простые подзадачи:
Если n=0, результат равен 1 (любое число в нулевой степени — 1).
Если n>0, используем свойство:
при чётном n: xn =(xn/2)2;
при нечётном n: xn=x⋅xn−1
3.Временная сложность
O(log n) — логарифмическая сложность.
4.Описание временной сложности (почему?)
На каждом шаге рекурсии показатель степени уменьшается вдвое (при чётном n) или на единицу (при нечётном n). В худшем случае (когда n — степень двойки) количество вызовов равно количеству битов в двоичной записи n, то есть log 2n. 
Даже при нечётных n после одного шага вычитания получается чётное число, и далее степень быстро сокращается вдвое.

10. Какая разница между обходом дерева в глубину (DFS) и в ширину (BFS)?
Порядок обхода:
DFS (Depth‑First Search, в глубину) — идёт «вглубь» по одной ветви до конца, затем возвращается и исследует следующие ветви.
BFS (Breadth‑First Search, в ширину) — обходит дерево «по уровням»: сначала корень, затем все узлы уровня 1, затем уровня 2 и т. д.
Структура данных:
DFS — обычно реализуется через стек (рекурсия или явный стек).
BFS — использует очередь (FIFO).
Когда применять:
DFS — когда нужно дойти до листа/цели по одному пути, обход поддеревьев, топологическая сортировка.
BFS — когда важен кратчайший путь от корня, обход по уровням, поиск в неориентированных графах.
Память (в среднем):
DFS — потребляет память пропорционально максимальной глубине дерева.
BFS — потребляет память пропорционально ширине самого широкого уровня.
