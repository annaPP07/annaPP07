Вариант 18. Табу-поиск для раскраски графа
Задача: реализовать табу-поиск для задачи раскраски графа.
Требования:
- Использовать табу-лист для хранения последних изменений цветов
- Итерации поиска: 500
- Размер табу-листа: 10
- Вывести финальное количество цветов и распределение
Входные данные:
Граф с 12 вершинами

Алгоритм
1.Инициализация:Генерируем начальную раскраску (например, случайным образом или жадным алгоритмом). Устанавливаем счётчик итераций и инициализируем табу‑список (хранит последние изменения: пара вершина → старый цвет).
2.Основной цикл (до 500 итераций):
 Для каждой вершины пробуем сменить цвет на допустимый (не конфликтующий с соседями).
 Выбираем ход с наибольшим уменьшением числа конфликтов (или минимальным ухудшением, если конфликтов нет).
 Если ход в табу‑листе — пропускаем.
 Применяем ход, обновляем раскраску и добавляем изменение в табу‑лист (если размер > 10 — удаляем старейшее).
3.Критерий остановки:
 Достигнуто 500 итераций.
 Найдено решение без конфликтов (оптимум).
4.Вывод:
 Количество использованных цветов.
 Распределение цветов по вершинам.
Временная сложность:
 Сложность: O(n2) на итерацию (где n — число вершин).

18. NP-сложность
Почему мы используем приближенные алгоритмы для NP-сложных задач? Объясните суть
проблемы.
NP-сложные задачи — это класс задач, для которых не известно полиномиальных алгоритмов, гарантирующих нахождение точного оптимального решения за разумное время. Большинство таких задач не могут быть решены эффективно даже на современных компьютерах, если не использовать перебор, который становится непрактичным уже при небольших размерах входных данных.​
Именно поэтому для NP-сложных задач часто используют приближенные алгоритмы. Их суть заключается в том, чтобы за полиномиальное время находить решения, которые не обязательно оптимальны, но гарантированно не сильно отличаются от оптимального (например, не хуже чем в два раза). Такие алгоритмы позволяют получать приемлемые результаты на практике, даже если точное решение найти невозможно.​
Таким образом, приближенные алгоритмы — это компромисс между качеством решения и временем его поиска, позволяющий решать NP-сложные задачи в реальных условиях.​
